How to create a blog using Github Pages and Hugo in 10 minutes

Aug 11, 2023

Step by step guide

Let’s say we want to create a new blog called test_site

Step 1: Set up 2 Github repos

  • test_site_blog: blog repo. This is the main repo where all the data assets and blogs are saved
  • test_site.github.io: production repo. Tths is we deploy the page from, it has to come with the suffix github.io in order to be published in Github properly

Step 2: install Hugo

brew install hugo

Step 3: create a Hugo side in blog repo

hugo new site test_site

Step 4: add themes

  • install color themes
git submodule add https://github.com/nanxiaobei/hugo-paper themes/paper
  • Edit hugo.toml file, each them has some examples of what you can customize, but below items are the most important ones
baseURL = 'https://richlin.github.io/'
languageCode = 'en-us'
title = 'Test'
theme = "paper"

Check the theme page here https://themes.gohugo.io/themes/hugo-paper/#options

Create first blog

hugo new posts/test.md

This will create the first blog post in the folder

Step 5: test website

hugo server

Step 6: connect 2 repos

  • Move to the richlin.github.io repo, edit readme file, and push to Github. You will need to have 1 commit for this to work properly.
git submodule add https://github.com/richlin/richlin.github.io.git public

Step 7

Make sure everything works fine.

hugo server

Generate static files

hugo -t paper

How to publish a post in the blog

Write your blog

  • You can simply add a blog post by creating a MD file under /contet/posts/ folder
  • After finish writing the post, need to generate static files again in Public folder, because the public folder is where the webpage links to.
hugo -t paper

Push to git

Here you will need to push code in 2 places

  • blog folder
git add .
git commit 
git push
  • public folder
cd public
git add .
git commit 
git push
