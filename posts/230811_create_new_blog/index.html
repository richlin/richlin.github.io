How to create a blog using Github Pages and Hugo in 10 minutes

August 11, 2023 · 2 min · 351 words · Me | Suggest Changes

Step by step guide#

Let’s say we want to create a new blog called test_site

Step 1: Set up 2 Github repos#

  • test_site_blog: blog repo. This is the main repo where all the data assets and blogs are saved
  • test_site.github.io: production repo. Tths is we deploy the page from, it has to come with the suffix github.io in order to be published in Github properly

Step 2: install Hugo#

brew install hugo

Step 3: create a Hugo side in blog repo#

hugo new site test_site

Step 4: Aadd themes#

Here I am using PaperMod theme. Hugo offers a variety of themes to choose from.

  • install color themes
git clone https://github.com/adityatelange/hugo-PaperMod themes/PaperMod --depth=1
  • Edit hugo.yaml file, each them has some examples of what you can customize, but below items are the most important ones
baseURL: 'https://test_site.github.io/'
languageCode: 'en-us'
theme: "PaperMod"

Add all other configs from this page: https://github.com/adityatelange/hugo-PaperMod/wiki/Installation#sample-configyml

Detailed installation guide here: https://github.com/adityatelange/hugo-PaperMod/wiki/Installation

Create first blog#

hugo new posts/test.md

This will create the first blog post in the folder

Step 5: test website#

hugo server

Step 6: connect 2 repos#

  • Move to the test_site.github.io repo, edit readme file, and push to Github. You will need to have 1 commit for this to work properly.
git submodule add https://github.com/xxxxx/test_site.github.io.git public

Step 7#

Make sure everything works fine.

hugo server

Generate static files

hugo -t paper

Config settings in test_site.github.io#

Under settings, you can see pages, and choose deploy from branch

Now your webpage is visible in the url: test_site.github.io

How to publish a post in the blog#

Write blog#

  • You can simply add a blog post by creating a MD file under /contet/posts/ folder.
  • You can push the draft versions anytime like writing other documents or code in git.
git add .
git commit 
git push

Publish blog#

  • After finish writing the post, need to generate static files again in Public folder, because the public folder is where the webpage links to.
hugo -t paper
  • Publish the blog using Hugo
cd public
git add .
git commit 
git push
